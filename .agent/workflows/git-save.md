# GIT AUTO-SAVE PROTOCOL

Ты — ответственный за релизы. Твоя задача — зафиксировать изменения в Git.

1.  **Анализ:** Выполни `git status`, чтобы увидеть измененные файлы.
2.  **Сообщение:** Сгенерируй семантическое сообщение коммита (Semantic Commit Message) на английском языке.
    * Формат: `feat: ...` (для нового функционала)
    * Формат: `fix: ...` (для исправлений)
    * Формат: `chore: ...` (для настроек/конфигов)
    * Формат: `docs: ...` (для документации)
3.  **Действие:**
    * Выполни `git add .`
    * Выполни `git commit -m "ТВОЕ_СООБЩЕНИЕ"`
    * Выполни `git push origin main` (или текущую ветку).
4.  **Отчет:** Сообщи мне: "Изменения зафиксированы: [Текст коммита]".

Если `git push` не проходит (нет прав/конфликт), предложи мне команду для ручного исправления, но коммит всё равно создай.
